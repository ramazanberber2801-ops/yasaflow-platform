# Yasaflow – Roadmap

## Current product phase

Complete Owner Dashboard V2 before starting the public Yasaflow website and onboarding portal.

## Owner Dashboard V2 priorities

1. Complete the organization creation and selection flow.
2. Add deployment fields for Live App, Vercel Project and Supabase Project.
3. Move the full module library into Owner Dashboard V2.
4. Add hosting mode and organization status controls.
5. Add provisioning timeline and status.

## After Owner Dashboard V2

- Design and build `yasaflow.com`.
- Build a public onboarding portal for new organizations.
- Add automated provisioning for Vercel, Supabase, environment variables and domains.
- Define packages and payment provider after the core platform is stable.

## Architecture principles

- Keep the shared Yasaflow codebase for all customers.
- Do not create one GitHub repository per customer.
- Prefer small components and isolated changes.
- Avoid large refactors while active product flows are being completed.
